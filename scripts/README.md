# WaffleFinance Scripts

This directory contains utility scripts for development, testing, and release management.

## Release Verification Scripts

### `verify-release-locally.sh` (Linux/macOS)

Bash script that runs comprehensive release verification checks locally before pushing a release tag.

**Prerequisites:**
- Node.js >= 22.0.0
- pnpm >= 8.0.0
- Foundry (forge)
- jq (for JSON parsing)

**Usage:**
```bash
chmod +x scripts/verify-release-locally.sh
./scripts/verify-release-locally.sh
```

### `verify-release-locally.ps1` (Windows)

PowerShell script that runs the same release verification checks on Windows.

**Prerequisites:**
- Node.js >= 22.0.0
- pnpm >= 8.0.0
- Foundry (forge)
- PowerShell 5.1 or later

**Usage:**
```powershell
.\scripts\verify-release-locally.ps1
```

## What These Scripts Do

Both scripts perform the following verification steps:

### 1. Contract Verification
- ✅ Compile contracts with Hardhat
- ✅ Verify Hardhat artifacts exist for all contracts
- ✅ Compile contracts with Foundry
- ✅ Verify Foundry artifacts exist
- ✅ Check bytecode consistency between Hardhat and Foundry
- ✅ Run Hardhat test suite
- ✅ Run Foundry fuzz and invariant tests
- ✅ Generate artifact checksums

### 2. SDK Package Verification
- ✅ Build SDK package
- ✅ Verify all build outputs exist
- ✅ Verify all package.json exports have corresponding files
- ✅ Test that all exports can be imported
- ✅ Run SDK test suite
- ✅ Check package size
- ✅ Generate build checksums

### 3. Full Workspace Verification
- ✅ Build all packages
- ✅ Typecheck all packages (SDK, coordinator, resolver, frontend)

### 4. Generate Report
- Display summary of all verification results
- Show artifact checksums
- Provide guidance on next steps

## Why Run These Scripts?

Running these verification scripts locally before pushing a release tag helps you:

1. **Catch issues early** - Find problems before they reach CI
2. **Save time** - Avoid failed CI runs and re-tagging
3. **Ensure quality** - Verify that all artifacts build correctly
4. **Gain confidence** - Know that your release will pass CI checks

## When to Run

Run these scripts:
- ✅ Before creating and pushing a release tag
- ✅ After making significant changes to contracts or SDK
- ✅ After updating dependencies
- ✅ When troubleshooting CI failures

## Interpreting Results

### Success ✅
If the script completes successfully, you'll see:
```
✅ All release verification steps passed! ✨

You can now safely push your release tag:
  git tag -a v1.0.0 -m 'Release version 1.0.0'
  git push origin v1.0.0
```

### Failure ❌
If any step fails, the script will:
1. Display an error message indicating which step failed
2. Exit immediately with a non-zero status code
3. You must fix the issue before proceeding

Common failure reasons:
- **Compilation errors** - Fix syntax or type errors in contracts
- **Test failures** - Fix failing tests
- **Missing artifacts** - Ensure build outputs are generated correctly
- **Type errors** - Fix TypeScript type issues
- **Import errors** - Fix package exports or dependencies

## Checksums

The scripts generate SHA-256 checksums for:

1. **Contract artifacts** - Hash of all compiled contract JSON files
2. **SDK build** - Hash of all built SDK files

These checksums:
- Ensure build reproducibility
- Help detect unintended changes
- Match the checksums generated by CI
- Can be used for post-release verification

## Troubleshooting

### "jq: command not found" (Bash script)
Install jq:
```bash
# macOS
brew install jq

# Ubuntu/Debian
sudo apt-get install jq

# Fedora
sudo dnf install jq
```

### "forge: command not found"
Install Foundry:
```bash
curl -L https://foundry.paradigm.xyz | bash
foundryup
```

### "pnpm: command not found"
Install pnpm:
```bash
npm install -g pnpm@8.15.0
```

### Script fails on Windows with "Execution of scripts is disabled"
Enable script execution in PowerShell:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Tests fail locally but pass in CI
Ensure your local environment matches CI:
- Check Node.js version: `node --version` (should be 22.x)
- Check pnpm version: `pnpm --version` (should be 8.15.0)
- Ensure dependencies are installed: `pnpm install`
- Clean build artifacts: `pnpm clean && pnpm install`
